intro

git mirroring is a process of replicating a repository from one location to another one. to ensure that there is multiple copies of the repository available for use when needed. these mirrors can be used as backups, or to ensure accessing repositories easily for large distributed teams.

of course git mirroring can be done manually by copying the repo files to another locations, but this approach will make it difficult to keep the copies in sync. (which is what git distribution is made for)

steps

to mirror a git repository using git mirror functionality internals :

  • create empty repository at the destination location
  • in source repo: git remote add mirror git://example.com/mirror.git
  • git remote -v to verify that the reference has been added
  • git push --mirror mirror

git hooks

at last we can use git push --mirror mirror to push everytime we make a change in the original repo or when we want to push, but if we want to make this process automatically happen when we push the repo to the src we can use git hooks : #!/bin/bash git push --mirror mirror and make it executable chmod +x mirroring.sh
